BitTorrent关键技术

grid_view

分块下载机制

  • content_cut
    文件分块 - 文件被分割为256KB固定大小的块
  • view_module
    切片传输 - 每个块再分为16KB的切片作为基本传输单位
  • shuffle
    随机策略 - 以随机顺序请求下载文件块，提高效率
  • priority_high
    稀有优先 - 优先下载拥有节点最少的分块，维持平衡
  • verified
    哈希验证 - 每个块下载后进行SHA1哈希验证确保完整性
message

消息类型

  • format_list_numbered
    消息格式 - <length prefix><type id><payload>
  • sync
    Keep-Alive - 每2分钟发送一次，维持连接

settings

状态机制

  • memory
    状态机 - 每个节点维护一个四比特位的状态机
本地状态
am_choking: 是否阻塞对方
am_interested: 是否对对方感兴趣
远程状态
peer_choking: 对方是否阻塞我
peer_interested: 对方是否对我感兴趣
  • download
    下载条件 - 对对方感兴趣且对方未阻塞
  • upload
    上传条件 - 对方感兴趣且未阻塞对方
balance

阻塞算法

  • block
    初始状态 - 对所有peer节点都是choked状态
  • timer
    定期评估 - 每10秒统计一次节点下载速度
  • speed
    最优选择 - 选择下载速度最快的4个节点解除阻塞
  • thumb_up
    兴趣条件 - 只对当前节点感兴趣的节点解除阻塞
  • swap_horiz
    公平机制 - 防止"只下载不上传"的自私行为
  • trending_up
    激励上传 - 上传越多，获得解除阻塞的机会越大
