第三部分：Git的引用机制

引用机制概述

引用(References)是Git中用于代替SHA-1值记忆的"别名"，存储在.git/refs/目录下

HEAD引用

指向当前检出的分支

  • 通常是一个符号引用
  • 指向其他引用而非直接指向对象
  • 执行git commit时作为父提交

示例：

ref: refs/heads/master

标签引用

标记特定提交的引用

  • 轻量级标签：直接指向Commit对象
  • 附注标签：创建Tag对象，包含更多信息
  • 永不改变，类似"固化的分支"

存储位置：

.git/refs/tags/

远程引用

远程仓库与本地仓库的映射

  • 记录远程服务器上分支的最后位置
  • 主要用于跟踪远程分支状态
  • 通常只读，不能直接通过commit更新

存储位置：

.git/refs/remotes/
