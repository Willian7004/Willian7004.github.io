Git实现原理详解

第二部分：Git的对象模型

对象模型概述

Git本质上是一个内容寻址的文件系统，所有内容都以对象形式存储

Blob对象

存储内容：

  • 文件内容数据
  • 不包含文件名和元数据

存储格式：

"blob" + 空格 + 内容字节数 + "\0" + 文件内容

示例：

sha1: fa49b077972391ad58037050f2a75f74e3671e92

Tree对象

存储内容：

  • 目录结构信息
  • 包含子目录(Blob)和子文件(Tree)的引用

存储格式：

"tree" + 空格 + 内容字节数 + "\0" + 树条目列表

树条目格式：

文件模式 + 空格 + SHA-1 + 空格 + 文件名

Commit对象

存储内容：

  • 版本索引和版本关系
  • 指向Tree对象
  • 父Commit对象引用
  • 作者、提交者信息
  • 提交信息

存储格式：

"commit" + 空格 + 内容字节数 + "\0" + 提交信息

Tag对象

存储内容：

  • 标记特定提交的引用
  • 包含标签名称、创建者信息
  • 附注标签包含详细注释

两种类型：

  • 轻量级标签：直接指向Commit对象
  • 附注标签：创建Tag对象，包含更多信息

对象关系与存储位置

Blob对象
.git/objects/
Tree对象
.git/objects/
Commit对象
.git/objects/
所有对象均通过SHA-1值进行唯一标识
