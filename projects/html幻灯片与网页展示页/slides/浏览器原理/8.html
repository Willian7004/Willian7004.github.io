浏览器性能优化策略

提升网页性能的关键技术

减少回流与重绘

回流优化

  • 使用transform替代top/left
  • 使用visibility替换display:none
  • 批量DOM操作，减少交互次数

重绘优化

  • 避免频繁修改复杂样式
  • 使用opacity替代visibility
  • 减少box-shadow等复杂属性 
// 不好的做法 - 触发多次回流
for (let i = 0; i < 100; i++) {
  element.style.width = (element.offsetWidth + 1) + 'px';
}

// 好的做法 - 批量操作
let width = element.offsetWidth;
for (let i = 0; i < 100; i++) {
  width++;
}
element.style.width = width + 'px';

执行优化

  • 使用防抖/节流控制高频事件
  • 避免长任务阻塞主线程
  • 使用requestAnimationFrame实现动画
  • 使用Web Worker处理复杂计算

CSS优化策略

选择器优化

  • 避免复杂选择器
  • 使用类选择器替代标签选择器
  • 避免过度层叠选择器

样式优化

  • 使用CSS变量减少重复代码
  • 避免@import，使用link标签
  • 压缩CSS文件减少体积

布局优化

使用flexbox/grid 避免table布局 使用CSS containment 减少布局层级 使用will-change属性

浏览器缓存优化

HTTP缓存策略

  • 设置合理的Cache-Control头
  • 使用ETag验证资源变更
  • 实现内容哈希文件名

存储优化

  • 使用Service Worker缓存资源
  • 合理使用localStorage和IndexedDB
  • 实现离线优先策略

资源加载优化

压缩资源文件 优化图片格式/大小 字体文件按需加载 代码分割与懒加载 预加载关键资源

关注FPS、FCP、LCP等核心指标
