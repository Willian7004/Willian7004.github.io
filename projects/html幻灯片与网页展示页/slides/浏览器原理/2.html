浏览器基本架构

多进程模型及其职责分工

Browser Process

  • 界面控制（地址栏、按钮）
  • 用户交互处理
  • 网络请求管理
  • 文件访问控制
  • 子进程协调与管理

Renderer Process

  • 网页内容渲染
  • DOM/CSS解析
  • JavaScript执行
  • 每个标签页一个进程

GPU Process

  • 图形处理加速
  • Canvas渲染
  • GPU资源管理
  • 跨进程共享

Plugin Process

  • 插件独立运行
  • Flash/ActiveX等
  • 沙箱隔离
  • 崩溃隔离

进程间通信 (IPC)

通信机制

  • 管道 (Pipes)
  • 消息队列 (Message Queues)
  • 共享内存 (Shared Memory)
  • 套接字 (Sockets)

优势

  • 稳定性：进程隔离，一个崩溃不影响其他
  • 安全性：权限隔离，限制恶意代码影响
  • 性能：资源独立，可根据设备调整
  • 可维护性：模块化设计，易于扩展
