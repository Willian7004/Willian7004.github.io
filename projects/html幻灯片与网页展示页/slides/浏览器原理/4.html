浏览器渲染引擎工作原理(一)

DOM构建与CSSOM构建

DOM构建

构建过程

1

词法分析 (Lexing)

将HTML字符流转换为Token序列

2

语法分析 (Parsing)

将Token转换为节点对象

3

构建DOM树

将节点连接成树形结构

DOM树示例

document
html
head
title
body
h1
p

HTML代码示例

<!DOCTYPE html>
<html>
  <head>
    <title>页面标题</title>
  </head>
  <body>
    <h1>标题</h1>
    <p>段落内容</p>
  </body>
</html>

CSSOM构建

构建过程

1

CSS解析

将CSS文本转换为样式规则

2

规则匹配

将选择器与DOM元素匹配

3

构建CSSOM树

形成层叠样式表对象模型

CSSOM树示例

document
html
body
h1 { color: blue; }
p { font-size: 16px; }

CSS代码示例

body {
  font-family: Arial, sans-serif;
  margin: 0;
  padding: 20px;
}

h1 {
  color: blue;
  font-size: 24px;
}

p {
  font-size: 16px;
  line-height: 1.5;
}

关键点

阻塞渲染：CSSOM构建会阻塞DOM构建和页面渲染

继承与层叠：样式规则通过继承和层叠确定最终样式

性能影响：CSSOM构建是资源密集型操作
