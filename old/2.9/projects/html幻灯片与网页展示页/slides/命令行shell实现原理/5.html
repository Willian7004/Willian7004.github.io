命令执行（fork-exec模型）

fork-exec模型

父进程 (Shell)
- 等待用户输入
- 解析命令
- 创建子进程
- 等待子进程结束
子进程
- 复制父进程内存空间
- 执行新程序
- 退出（exec失败时）

关键技术点

fork()系统调用
创建子进程，返回两次：
- 父进程：返回子进程PID
- 子进程：返回0
execvp()函数族
- v：参数以数组形式传递
- p：自动搜索PATH环境变量
waitpid()同步机制
- 阻塞等待子进程结束
- 防止僵尸进程产生
代码示例
// 命令执行引擎
int Execute() {
    pid_t id = fork();
    if (id == 0) {  // 子进程
        // 执行程序替换
        execvp(g_argv[0], g_argv);
        exit(1);  // exec失败时退出
    }
    // 父进程
    waitpid(id, NULL, 0);
    return 0;
}
